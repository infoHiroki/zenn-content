# お前はもう日報を書いている

前回の記事で、俺は日記アプリを全部捨てた話をした。

👉 [Notion → Obsidian → Logseq → 結局これ](https://zenn.dev/infohiroki/articles/journaling-tool-evolution)

今回は、その続きだ。**tower**というシステムを作った話。

毎朝「おはよう」って言うだけで、AIが日記の準備をしてくれるやつだ。

**GitHubリポジトリ**: [tower-template](https://github.com/infohiroki/tower-template)

---

## 日記アプリへの恨みつらみ

### なぜ日記アプリは続かないのか

日記アプリ、何個試した？

俺は数えきれない。Notion、Obsidian、Logseq、Day One、Journey、Grid Diary...全部ダメだった。

なぜか。理由は3つ。

1. **複雑すぎる** - 機能が多すぎて開く気が失せる。テンプレート？タグ？リマインダー？うるさい。
2. **アプリを開くのが面倒** - 「日記書くぞ」という意志が必要。意志なんてものは朝のコーヒーが冷める前に消える。
3. **データが人質** - サービス終了したら？お前のデータ、どこいくの？

### Markdown + Git = 自由

結論から言う。

**Markdownファイルをテキストエディタで開いて、Gitでコミットする**。

これだけ。これだけでいい。

- Markdownは10年後も読める
- Gitは変更履歴が全部残る
- どのマシンからでも同期できる
- サービス終了？ない。自分のファイルだから。

でも、これだけだと面倒なんだ。毎日ファイル作って、昨日の振り返りして、習慣のストリーク数えて...

そこでClaude Codeの出番だ。

---

## towerとは

### 「おはよう」だけでいい

朝起きて、ターミナル開いて、Claude Codeで「おはよう」って言う。

するとAIが：

1. 今日の日記ファイルを作る
2. 昨日の習慣を確認してストリークを計算
3. 昨日の「気づき」「心がけ」を読み上げる
4. 今日へのアドバイスをくれる

俺がやることは「おはよう」と言うだけ。

日記を「書く」作業に集中できる。準備はAIがやる。

## どうなってるのか

### ディレクトリ構造

```
tower/
├── CLAUDE.md              # AIへの指示書
├── diary/
│   ├── template.md        # 日記テンプレート
│   └── 2026/02/03.md      # 今日の日記
└── scripts/
    └── sync-repos.clj     # おまけ
```

シンプルだろ？

`CLAUDE.md`にルールを書いておくと、Claude Codeがそれに従って動く。「おはよう」と言われたら何をするか、全部ここに書いてある。

## 習慣トラッキング

### ストリークの魔力

```markdown
## 習慣
- [ ] 運動する 🔥5
- [ ] 読書する 🔥12
- [ ] 瞑想する 🔥3
```

この`🔥12`が見えるか。

12日連続で読書してる。この数字を見ると、途切れさせたくない。明日は`🔥13`にしたい。

人間は弱い。意志の力なんて信用するな。でも数字には弱い。ゲーミフィケーションの力を借りろ。

AIが前日の日記を読んで、達成してたらストリークを+1する。達成してなかったら0に戻す。全自動。

### Never miss twice

**1日サボるのはOK。2日連続はNG。**

これだけ覚えとけ。

完璧主義は習慣の敵だ。「今日できなかった...もうダメだ」って思うな。1日くらい許せ。

ただし、2日目は何があってもやれ。2日連続でサボると、習慣は死ぬ。

## 日記テンプレート

### 感謝・気づき・心がけ

```markdown
## 感謝
今日感謝できることは？
- 朝起きれたこと
- コーヒーが美味かったこと

## 気づき
- 午前中が一番集中できる

## 心がけ
明日意識したいこと：
- 夜更かししない
```

これを毎日書く。

そして翌朝、AIが「昨日の気づき」と「心がけ」を読み上げてくれる。

「昨日、あなたは『夜更かししない』と書きました。今日も意識しましょう」

自分で書いた言葉を、翌朝自分に返されると、なんか効く。

### 自己評価

```markdown
## 自己評価
**今日の評価**: ⭐⭐⭐⭐
```

1日の終わりに星をつける。

⭐⭐⭐⭐⭐ = 最高
⭐ = 最悪

良し悪しじゃない。「この日はどうだったか」を記録するだけ。あとで振り返るときに役立つ。

## Claude Codeとの連携

### CLAUDE.mdの中身

```markdown
## 朝のルーティン（おはよう）

ユーザーが「おはよう」と言ったら：
1. diary/template.md を元に今日の日記ファイルを作成
2. 前日の習慣チェックボックスを確認してストリークを更新
3. 前日の「気づき」「心がけ」を読み上げ
4. 今日に活かせるポイントを1つ提案
```

これをCLAUDE.mdに書いておく。Claude Codeはこれを読んで、その通りに動く。

カスタマイズしたければ、このファイルをいじればいい。

### MCPでさらに自動化

MCPサーバーを使えば、もっと色々できる。

- **google-calendar** → 今日の予定を日記に自動挿入
- **github** → 昨日のコミット履歴を自動ログ

でもなくても動く。シンプルに始めろ。

## セットアップ

### 5分で終わる

```bash
# 1. テンプレートをフォーク
# GitHub で「Use this template」をクリック

# 2. クローン
git clone https://github.com/YOUR_USERNAME/tower.git ~/tower

# 3. 開く
cd ~/tower
claude

# 4. 言う
おはよう
```

これだけ。

`diary/template.md`を自分好みにいじれ。習慣リストを変えろ。追加したい項目があれば追加しろ。

## まとめ

日記が続かないのは、お前の意志が弱いからじゃない。

**仕組みがダメ**だからだ。

- アプリを開くな → ターミナルで完結させろ
- 準備するな → AIにやらせろ
- 完璧を目指すな → Never miss twice

towerはそういう仕組みだ。

日記や習慣管理で100回挫折した人間が、やっと続いてる方法。試してみてくれ。
