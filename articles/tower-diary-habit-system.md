# お前はもう日報を書いている 🗼

この記事を読み終わる頃には、お前は日報を「書く」という発想を捨てている。

俺は日記アプリを全部捨てた。そして**喋る**ことにした。

👉 [前回: Notion → Obsidian → Logseq → 結局これ](https://zenn.dev/infohiroki/articles/journaling-tool-evolution)

今回は、その続きだ。**tower**というシステムを作った話。

「おはよう」って喋るだけで、日記の準備が終わる。そして喋ってるうちに、日報も書かれてる。

**GitHubリポジトリ**: [tower-template](https://github.com/infohiroki/tower-template)

---

## 🎤 喋れば日記になる

俺はAqua Voiceを使ってる。音声入力だ。

Claude Codeを開いて、喋る。

> 「今日は朝から頭痛がひどくて、でも午後には回復して、結局3時間くらいコード書いた」

これだけ。Claude Codeが勝手に整理してログに残してくれる。

タイピング？ **しない。**
アプリを開く？ **ターミナルは開いてる。**
書くぞという意志？ **いらない。喋ってるだけ。**

だから「お前はもう日報を書いている」。

**喋ってる時点で、もう書いてる。** ✍️

---

## 🔥 Gitを見れば、何やったか分かる

もう一つの武器がある。Git。

コード書いてコミットしてれば、その日の活動は全部残ってる。

「今日何やったっけ」→ Gitのコミット履歴を見ればいい。

Claude Codeなら:
- 🔗 MCPでGitHub連携してれば自動で取得
- 💬 してなくても「今日のコミット見て」で取れる

つまり:
- **喋った内容** → ログ 📝
- **Gitのコミット** → 活動記録 💻

この2つが揃えば、日報は自動で完成する。

「今日のまとめ書いて」って言えば、Claude Codeが整理してくれる。

お前は何も書いてない。喋って、コード書いて、それだけ。**でも日報はできてる。** 🎉

---

## 😤 日記アプリへの恨みつらみ

### なぜ日記アプリは続かないのか

日記アプリ、何個試した？

俺は数えきれない。Notion、Obsidian、Logseq、Day One、Journey、Grid Diary...全部ダメだった。

なぜか。理由は3つ。

1. **複雑すぎる** 🤯 - 機能が多すぎて開く気が失せる。テンプレート？タグ？リマインダー？うるさい。
2. **アプリを開くのが面倒** 😩 - 「日記書くぞ」という意志が必要。意志なんてものは朝のコーヒーが冷める前に消える。
3. **データが人質** 🔒 - サービス終了したら？お前のデータ、どこいくの？

### Markdown + Git = 自由 🕊️

結論から言う。

**Markdownファイルをテキストエディタで開いて、Gitでコミットする**。

これだけ。これだけでいい。

- ✅ Markdownは10年後も読める
- ✅ Gitは変更履歴が全部残る
- ✅ どのマシンからでも同期できる
- ✅ サービス終了？ない。自分のファイルだから。

でも、これだけだと面倒なんだ。毎日ファイル作って、昨日の振り返りして、習慣のストリーク数えて...

そこでClaude Codeの出番だ。 🤖

## 🗼 towerとは

### 「おはよう」だけでいい ☀️

朝起きて、ターミナル開いて、Claude Codeで「おはよう」って喋る。

するとAIが：

1. 📄 今日の日記ファイルを作る
2. 🔥 昨日の習慣を確認してストリークを計算
3. 💭 昨日の「気づき」「心がけ」を読み上げる
4. 💡 今日へのアドバイスをくれる

俺がやることは「おはよう」と喋るだけ。

日記を「書く」なんて意識しなくていい。準備はAIがやる。

## 📁 どうなってるのか

### ディレクトリ構造

```
tower/
├── CLAUDE.md              # これがすべて（ルール＝テンプレート）
└── diary/
    ├── template.md        # 初日用（参考）
    └── 2026/
        └── 02-03.md       # 今日の日記
```

`template.md`？ **いらない。** 🚫

CLAUDE.mdにルールが書いてあれば、**前日の日記をコピー**して作る。使ってるうちに日記の形は勝手に進化する。

`CLAUDE.md`にルールを書いておくと、Claude Codeがそれに従って動く。「おはよう」と言われたら何をするか、全部ここに書いてある。

## 🔥 習慣トラッキング

### ストリークの魔力 ✨

```markdown
## 習慣
- [ ] 運動する 🔥5
- [ ] 読書する 🔥12
- [ ] 瞑想する 🔥3
```

この`🔥12`が見えるか。

12日連続で読書してる。この数字を見ると、途切れさせたくない。明日は`🔥13`にしたい。

人間は弱い。意志の力なんて信用するな。でも数字には弱い。**ゲーミフィケーションの力を借りろ。** 🎮

AIが前日の日記を読んで、達成してたらストリークを+1する。達成してなかったら0に戻す。全自動。

### Never miss twice 🛡️

**1日サボるのはOK。2日連続はNG。**

これだけ覚えとけ。

完璧主義は習慣の敵だ。「今日できなかった...もうダメだ」って思うな。1日くらい許せ。

ただし、2日目は何があってもやれ。2日連続でサボると、習慣は死ぬ。 ☠️

## 📝 日記の中身

### 感謝・気づき・心がけ 🙏

```markdown
## 感謝
今日感謝できることは？
- 朝起きれたこと
- コーヒーが美味かったこと

## 気づき
- 午前中が一番集中できる

## 心がけ
明日意識したいこと：
- 夜更かししない
```

これを毎日書く。というか**喋る**。 🎤

そして翌朝、AIが「昨日の気づき」と「心がけ」を読み上げてくれる。

> 「昨日、あなたは『夜更かししない』と書きました。今日も意識しましょう」

自分で書いた言葉を、翌朝自分に返されると、なんか効く。 💪

### 自己評価 ⭐

```markdown
## 自己評価
**今日の評価**: ⭐⭐⭐⭐
```

1日の終わりに星をつける。

⭐⭐⭐⭐⭐ = 最高
⭐ = 最悪

良し悪しじゃない。「この日はどうだったか」を記録するだけ。あとで振り返るときに役立つ。

## 🤖 Claude Codeとの連携

### CLAUDE.mdの中身

```markdown
## 朝のルーティン（おはよう）

ユーザーが「おはよう」と言ったら：
1. **前日の日記をコピー**して今日のファイルを作成
2. チェックボックスをリセット、完了タスクを削除
3. ストリークを計算して更新
4. 前日の「気づき」「心がけ」を読み上げ
5. 今日に活かせるポイントを提案
```

シンプルだろ？ 😎

そしてこのCLAUDE.md自体も、Claude Codeに「このルール追加して」と言えば更新してくれる。

**設定すらAIが管理する。** 🧠

### なぜテンプレートじゃなくコピーなのか 🤔

日記は進化する。

使ってるうちに「この項目いらない」「これ追加したい」ってなる。テンプレートファイルを毎回編集するのは面倒。

前日コピーなら、昨日の自分が「ベストな形」を作ってくれてる。習慣リストもタスクも、そのまま引き継げる。

**昨日の自分がテンプレートだ。** 📅

### MCPでさらに自動化 🔌

MCPサーバーを使えば、もっと色々できる。

- **google-calendar** → 今日の予定を日記に自動挿入 📅
- **github** → 昨日のコミット履歴を自動ログ 💻

でもなくても動く。シンプルに始めろ。

## 🚀 セットアップ

### 5分で終わる ⏱️

```bash
# 1. テンプレートをフォーク
# GitHub で「Use this template」をクリック

# 2. クローン
git clone https://github.com/YOUR_USERNAME/tower.git ~/tower

# 3. 開く
cd ~/tower
claude

# 4. 喋る
おはよう
```

これだけ。

最初の日記だけ自分で作れ。習慣リストを書いて、その日のことを書く。翌朝から、AIがコピーして準備してくれる。

使ってるうちに、日記の形は勝手に最適化されていく。 ✨
