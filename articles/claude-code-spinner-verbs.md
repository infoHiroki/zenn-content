## 😵 Thinking...に飽きた

Claude Codeの待ち時間。

「Thinking...」「Reading file...」「Writing code...」

毎日何時間も見てる。ツールを呼ぶたびにスピナーが回って、英語の動名詞がぐるぐる流れていく。内容はまあ分かる。分かるけど飽きた。

ある日、[multi-agent-shogun](https://github.com/yohey-w/multi-agent-shogun)（tmuxベースのマルチエージェントシステム）のコードを読んでて、`settings.json`に面白い設定を見つけた。

`spinnerVerbs`。

スピナーのテキストを自由に書き換えられる。日本語もいける。何個でも入る。

「Thinking...」じゃなくて好きな言葉を表示できる。

こういうの大好き 🔥

## 🎰 まず設定方法（30秒で終わる）

`~/.claude/settings.json` を開いて、これを足すだけ:

```json
{
  "spinnerVerbs": {
    "mode": "replace",
    "verbs": [
      "般若心経ラップ中",
      "了解です（了解してない）",
      "脳内会議が白熱中"
    ]
  }
}
```

以上。設定後すぐに反映される 🎉

![VSCode + Claude Codeでスピナーが動いている作業風景](/images/spinner-work.png)
*実際の作業画面。カスタムスピナーが処理中に表示されている*

- `mode: "replace"` でデフォルトの英語を完全に置き換え
- `mode: "append"` だとデフォルトに追加
- `verbs` に文字列の配列を突っ込む。日本語OK。何個でもOK

3個でもいい。1000個でもいい。Claude Codeは処理中にこの中からランダムで1つ表示する。

これだけ。簡単だろ？

問題はここからだ。「3個でいいや」とはならなかった。

## ⚡ ここからが本番 — 待ち時間を実用化する

楽しいだけじゃ終わらせなかった。

ある日思った。**「ここにタスク載せたらいいんじゃないの？」** 💡

Claude Codeの待ち時間、結構長い。数秒から数十秒、ときには数分。その間ずっとスピナーのテキストが目に入る。

だったら、忘れたくないことをスピナーに載せたらいいじゃないか。

### 🏯 Towerシステムとの連携

俺は毎朝「Tower」っていう日記＋習慣管理システムで1日を始める。Markdownの日記に今日のタスクを書いて、習慣トラッカーをチェックして、カレンダーの予定を確認する。

この日記のデータをスピナーに食わせた。

`spinner-gen.sh`というスクリプトを書いて、5つのソースからスピナー動詞を動的に生成する:

| ソース | 内容 | 例 |
|--------|------|------|
| 🎨 楽しい動詞 | 535個のfun-verbs.json | 「般若心経ラップ中」 |
| ✅ 日記のタスク | 未完了タスク | 「記事を書く」 |
| 📅 カレンダー | 今日の予定 | 「15:00 MTG」 |
| 🔥 習慣トラッカー | 未達成の習慣 | 「運動する 🔥2」 |
| 🧠 アイデンティティ | 自分のあり方 | 「哲学的にデバッグ中」 |

### ⏰ 時間帯で自動切替

`auto`プリセットにすると、時間帯で配分が自動的に変わる:

| 時間帯 | 🎨 楽しい | ✅ タスク | 📅 予定 | 🔥 習慣 | 🧠 自分 |
|--------|--------:|--------:|-------:|-------:|-------:|
| 🌅 朝（5-9時） | 10% | 0% | 30% | 40% | 20% |
| ☀️ 昼（9-18時） | 0% | 60% | 30% | 10% | 0% |
| 🌆 夕（18-22時） | 40% | 25% | 15% | 10% | 10% |
| 🌙 夜（22-5時） | 70% | 0% | 0% | 10% | 20% |

- **朝**: 習慣を思い出せ。「運動する 🔥2」← 2日連続してるぞ、今日もやれ 💪
- **昼**: 仕事のタスクを表示。「APIリファクタ」「テスト書く」。忘れないように
- **夕**: ミックス。仕事のタスクも残ってるけど、楽しい動詞も混ぜる
- **夜**: ほぼ楽しい動詞。「般若心経ラップ中」の時間

土日はfun動詞が+20%される。仕事の日と休みの日で雰囲気が変わる。

### 🎯 待ち時間がリマインダーになった

これが地味に効く。

Claude Codeを使ってるだけで、今日やるべきことが目に入る。スピナーに「15:00 MTG」って表示されたら「あ、MTGあるんだった」って思い出す。「運動する 🔥3」って出たら「3日連続してるのに今日サボるのか？」って自分を追い込める。

**待ち時間が受動的なリマインダーになった**。意識しなくても情報が目に入る。これが一番の収穫 ✨

## 🐛 たまにバグる（それもまた良し）

正直に言う。スピナー、たまに文字化けする 🫠

全テキストがゴミ文字列になる。再現条件は不明。別のペインに移って戻ると治る。原因はまだ調査中。

でもバグってるスピナーも、それはそれで味がある。「卍解まであと少し」が「◆▼♦▲●■」に化けたら、むしろ卍解に近づいてる感じがする 😂

## 🔧 spinner-gen.shの仕組み

使い方はシンプル:

```bash
# autoプリセット（時間帯で自動切替）
./spinner-gen.sh --preset auto

# 楽しい動詞だけ
./spinner-gen.sh --preset fun

# タスク重視
./spinner-gen.sh --preset focus

# ドライラン（settings.jsonを更新せず表示だけ）
./spinner-gen.sh --preset auto --dry-run
```

プリセット:
- **fun**: 楽しい動詞のみ。遊びたい日
- **focus**: タスク60% + カレンダー30% + 習慣10%。集中したい日
- **mixed**: バランス型。fun40% + タスク25% + その他
- **auto**: 時間帯で重みを自動切替。おすすめ 🌟

内部では`jq`で各ソースからランダムに選んでマージ・シャッフルしてる。日記からタスクを`grep`して、カレンダーの予定を抜いて、習慣トラッカーの未達成を拾って、全部JSONの配列にして `settings.json` の `spinnerVerbs.verbs` に書き込む。

```bash
# settings.json更新の核心部分
jq --argjson verbs "$verbs" \
  '.spinnerVerbs.verbs = $verbs' \
  "$SETTINGS" > "$tmp"
mv "$tmp" "$SETTINGS"
```

不足分は楽しい動詞で補完される。タスクが0個の日でもスピナーは寂しくならない。

## 🏁 まとめ

Claude Codeの待ち時間、結構ある。毎日何時間分か「Thinking...」を見てる。

ここを自分の好きなもので埋める。それだけで開発体験が変わる。

やったこと:
1. **settings.jsonの設定を知る** — 30秒で終わる
2. **楽しい動詞を535個作る** — 止まらなくなる（副作用）
3. **きのこ動詞を501個作る** — 趣味が暴走する（副作用）
4. **待ち時間をリマインダーにする** — これが本命

最後が一番大事。楽しいだけじゃなく、実用的になった。待ち時間が、忘れたくないことを思い出させてくれる時間に変わった。

「Thinking...」を見てる時間がもったいないと思ったら、まず3個くらい好きな文を入れてみてくれ。止まらなくなるから 🎰

……ここに広告が出るようになったら終わりだけどな。「プランのアップグレードを検討中...」とか表示されたら俺はClaude Codeをやめる。
