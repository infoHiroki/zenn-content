俺はずっとChatGPTにコードを書かせてはコピペして、エラーが出たらまたChatGPTに貼って、を繰り返していた。

毎日これだ。コピペ、エラー、コピペ、エラー。まるで無限回廊にいるような気分。しかも途中でコンテキストが切れて「さっきのコードなんだっけ」とか言い出す。お前が書いたんだろうが。

そんな地獄を変えたのがClaude Codeだった。

## 🔥 コピペ地獄の終わり

Claude Codeは**ターミナルに住んでるAIエージェント**だ。チャットでコードを生成して人間がコピペする、みたいな話じゃない。こいつは自分でファイルを読み、コードを書き、実行し、エラーが出たら自分で直す。

人間がやることは「こういうの作って」と言うだけ。

大げさに聞こえるだろ？ 俺も最初はそう思った。でもマジだ。

![Claude Code デモ](https://github.com/anthropics/claude-code/raw/main/demo.gif)

## 📈 AIコーディングツール、3つの時代

俺がこの変化を実感するために、ちょっと歴史を振り返らせてくれ。

### 第1世代：コピペの時代 😵

ChatGPTに「こういう関数書いて」と頼む。書いてくれる。コピペする。動かない。エラー文をまた貼る。直してくれる。コピペする。動かない。永遠にこれ。

**問題は明白だった。** 全体のコードを見てないから、ファイルAを直したらファイルBが壊れる、みたいなことが起きまくる。コンテキストが断絶してるんだ。

### 第2世代：AIエディターの時代 🤔

CursorやGitHub Copilotが出てきた。エディタの中にAIがいて、コードの補完をしてくれる。これは良かった。コピペ地獄からは脱出できた。

でもまだ足りない。プログラムを**実行**するのは人間だった。環境構築も人間。テストも人間。「AIは補助、主役は人間」という構造は変わらなかった。

### 第3世代：AIエージェントの時代 🚀

で、Claude Codeが来た。こいつは違う。

- ✅ コマンドを自分で実行する
- ✅ エラーを自分で読んで自分で直す
- ✅ 環境構築からデプロイまで全部やる
- ✅ 「こういうアプリ作りたい」と言えば、まるっと作る

**主役が人間からAIに変わった。** 人間は方向を示すだけ。これが第3世代だ。

## 🔧 始め方（3行で終わる）

ごちゃごちゃ言わずにインストールしよう。

```bash
# macOS / Linux / WSL（これが一番楽）
curl -fsSL https://claude.ai/install.sh | bash
```

```bash
# Homebrewでも入る
brew install --cask claude-code
```

```bash
# npmでも（Node.js 18+が必要）
npm install -g @anthropic-ai/claude-code
```

インストールしたら `claude` とタイプするだけ。それだけでAIエージェントが立ち上がる。

VS CodeやCursorを使ってるなら拡張機能で「Claude Code」を検索してインストールすれば、エディタからも使える。

## 🎯 これだけ覚えろ5選

全部覚えようとするな。Claude Codeは機能が多すぎる。まずこの5つだけ覚えてくれ。

### 1. 対話モード 💬

```bash
claude
```

これで対話モードが始まる。やりたいことを日本語で言えばいい。「Reactでカウンターアプリ作って」でも「このバグ直して」でも何でもいい。

セッション管理も簡単だ。

```bash
claude --continue   # 直前のセッション続行
claude --resume     # 過去のセッション選択
```

**Ctrl+C** で抜ける。

### 2. /clear 🧹

```
/clear
```

**これが一番大事かもしれない。** 会話が長くなるとAIの精度が落ちる。新しいタスクに移る時は必ず `/clear` してくれ。

コンテキストウィンドウは有限だ。ゴミが溜まると判断がブレる。人間と同じだろ。

### 3. プランモード 📋

**Shift+Tab** を2回押すと「Plan Mode」に切り替わる。

いきなり「これ実装して」と言うな。まず計画を立てさせろ。

```
Yahooファイナンスから日本株データを取って
分析してバックテストするアプリを作りたい
```

こう言うと、Claude Codeが質問してくる。「UIはStreamlitでいい？」「パッケージ管理はuvでいい？」とか。要件を固めてから実装に入る。**急がば回れ**ってやつだ。

### 4. CLAUDE.md 📄

プロジェクトのルートに `CLAUDE.md` を置いておくと、Claude Codeが毎回それを読んでくれる。

```markdown
# プロジェクト設定

## 技術スタック
- FastAPI + HTMX
- Python 3.11
- UV でパッケージ管理

## コーディング規約
- 語尾に必ず「にゃ」をつけてください
```

↑最後のは冗談だが、実際にこういうプロジェクト固有のルールを書いておける。毎回同じことを説明しなくていい。

自動生成もできる。

```
/init
```

これでプロジェクトの状況を見ながら `CLAUDE.md` を作ってくれる。

⚠️ **注意**: 書きすぎるとコンテキストを圧迫して逆に精度が落ちる。重要なことだけ書け。

### 5. ultrathink 🧠

難しい実装の時はプロンプトの頭に「ultrathink」と入れろ。

```
ultrathink このリファクタリングの最適な方針を考えて
```

打った瞬間、文字が虹色になる。Claude Codeが本気出すサインだ。シンキングバジェットが最大になって、徹底的に考えてから答えを出してくれる。

ちなみに思考過程は **Ctrl+O** で覗ける。何を考えてるか見えるのはちょっと面白い。

## ⚡ MCPがヤバい

MCP（Model Context Protocol）は、Claude Codeに外部ツールを繋げる仕組みだ。これがまたヤバい。

### 何ができるか

- 🎭 **Playwright** でブラウザを自動操作してテスト
- 📋 **JIRA** のイシューから機能実装
- 🗄️ **PostgreSQL** にクエリ投げてデータ取得
- 🎨 **Figma** のデザインを読み込んでそのまま実装
- 📧 **Gmail** で下書き作成

### 設定は一言で終わる

```
claude
> プレイライトのMCPサーバーを設定して
```

Claude Codeが自分で調べて設定してくれる。マジで。

### Playwrightの実例

```
MCPが立ち上がっているので、PlaywrightのMCPを利用して
正しくバックテストできることを確認してください。
銘柄を複数選択できるようにして、動作確認するところまで進めてください。
```

こう言えばClaude Codeが自動でブラウザ立ち上げて、ポチポチ操作して、テスト結果を報告してくれる。俺はコーヒー飲んで待ってるだけ。

⚠️ MCPサーバーは有効にするとコンテキストをかなり食う。普段は `/mcp` でオフにしておいて、必要な時だけ有効化しろ。

## 🧩 プラグインとスキル

Claude Codeには拡張機能がいくつかある。

| 種類 | 何ができるか |
|------|-------------|
| 🛠️ **Custom Commands** | プロンプトを事前登録して呼び出し |
| 🤖 **Sub Agents** | 別人格のClaude（FE特化、データ分析特化とか） |
| 🪝 **Hooks** | コード作成後に必ず走る処理を設定 |
| 📚 **Skills** | 特定タスクの知識をパッケージ化 |
| 🔗 **MCP** | 外部ツール連携（上で説明した） |

### Context7 vs Skill

知識を拡張する方法が2つある。**Context7**と**Skill**だ。使い分けがちょっとだけ大事。

| | Context7（MCP） | Skill |
|---|---|---|
| 📝 **やること** | 最新のライブラリドキュメントをリアルタイム取得 | プロジェクト固有の知識をパッケージ化 |
| 🔄 **更新** | 常に最新 | 手動 |
| 📊 **コンテキスト消費** | デカい（必要時のみON） | 小さい（常時OK） |

**ざっくり言うと：**

- 「SvelteKit 2の最新ルーティング調べて」→ **Context7**
- 「うちのプロジェクトのコーディング規約で書いて」→ **Skill**

組み合わせるのが最強だ。

```
ultrathink Context7でSvelteKit 2の最新ルーティングを調べて、
my-project-styleスキルに従って実装してください
```

**最新の技術情報 + プロジェクトのルール**。これで出てくるコードの質が全然違う。

Skillは `.claude/skills/` にMarkdownファイルを置くだけで自作できる。

でも$20で「自分専属のジュニアエンジニア」が手に入ると思えば安い。コピペ地獄に戻りたいか？ 俺は嫌だ。

$100のMax 5xは、Opus 4.5という上位モデルが使えるのがデカい。Sonnetとの精度差は結構ある。本業で使うならこっち。

## 🎓 おまけ：学習にも使える

全部AIに任せるだけが使い方じゃない。

`CLAUDE.md` にこう書いてみろ。

```markdown
## 出力ガイド

実装はしないでください。
どういう変更が必要なのかと、なぜその変更をするのかを説明してください。
```

すると、コードを書く代わりに「このファイルのこの部分をこう変える。理由はこう」と教えてくれる。写経よりよっぽど身になる勉強法だ。

## 🔄 便利な小技

最後にいくつか小技を。

**許可スキップモード** 🏃（毎回「許可しますか？」がウザい人向け）

```bash
alias cc="claude --dangerously-skip-permissions"
```

⚠️ 名前の通りdangerousだ。Dockerとかで隔離した環境で使え。

**/rewind** ⏪（Escキー2回でも可）

Claude Codeが変な方向に行った時、特定のポイントまで巻き戻せる。ゲームのセーブポイントみたいなもんだ。

**画像入力** 📸

チャット欄にスクショを **Ctrl+V** で貼れる。「このデザインと同じものを作って」が一発で通る。言語化が面倒な時に最高。

**割り込み入力** ✋

AIが作業中でも入力できる。「ちょっと違うな」と思ったらその場で軌道修正。待たなくていい。

## 🏁 使う前と使った後

**Before:**
ChatGPTにコード生成→コピペ→エラー→コピペ→エラー→「もういい自分で書く」→3時間溶ける

**After:**
「こういうアプリ作って」→ 計画確認 → 承認 → コーヒー飲んで待つ → 動くものが出てくる

誇張じゃない。実際にこのサイト（infohiroki.com）のGo版からSvelteKit版への移植も、Claude Codeが大半のコードを書いた。102記事のMarkdown移行、多言語対応、SEO設定、全部だ。

最初は「AIに仕事を奪われる」とか思ってた。違った。**面倒な作業を奪ってくれた**んだ。俺はアイデアを考えることに集中できるようになった。

それだけで全部変わった。
